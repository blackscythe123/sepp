STUDENT MANAGEMENT SYSTEM

Practical Examination Question (Part C Implementation)

Scenario:

A college needs a software system to manage student records. The registrar should be able to add, view, search, and delete student information. The system must maintain an audit log of all operations.

Requirements:

Functional Requirements:

  • Add new student with ID, Name, Email, and Department
  • View all students in a list/table format
  • Search student by ID or Name
  • Delete student record with confirmation
  • Update student department
  • Prevent duplicate Student IDs
  • Validate email format
  • Log all operations (add, delete, search) with timestamp

Non-Functional Requirements:

  • All operations must complete within 30 seconds
  • Support up to 5000 student records
  • Auto-logout after 15 minutes inactivity
  • Clear error messages for invalid operations
  • Display confirmation dialogs for delete operations

Implementation Guidelines:

  1. Build using HTML and Vanilla JavaScript (NO CSS)
  2. Implement CRUD operations
  3. Use in-memory storage (arrays/objects)
  4. Include input validation
  5. Display results in table format
  6. Show success and error messages
  7. Create an activity log section

Deliverables:

Create a working HTML application that implements all the above requirements. The application should:

  • Have separate sections for Add, View, Search, Delete, Update operations
  • Validate all inputs before processing
  • Display clear feedback messages
  • Maintain activity log of all operations
  • Be fully functional and error-free

Code Template Provided:

Check the index.html file in this folder for a working implementation template.

Marks Distribution (Total: 20 marks):

  • User Interface & Usability: 4 marks
  • Add/Create Functionality: 3 marks
  • View/Search Functionality: 3 marks
  • Update/Delete Functionality: 3 marks
  • Validation & Error Handling: 4 marks
  • Activity Logging: 2 marks
  • Code Quality & Comments: 1 mark

Additional Challenge Questions (Recommended):

  • Add three subject marks per student and compute CGPA automatically using average/10
  • Generate department-wise report with student count and average CGPA
  • Classify students as Excellent, Good, or Needs Improvement based on average marks

Time Limit: 60 minutes

Language: HTML + JavaScript (No CSS allowed)
