FREELANCE JOB PORTAL

Practical Examination Question (Part C Implementation)

Scenario:

A startup wants to build a simple job board for freelancers. Clients should be able to post jobs with budgets, and freelancers should be able to browse these jobs and apply. The system must track all job postings and applications.

Requirements:

Functional Requirements:

  • Post a new job with Job Title, Budget (Rs), and Skills Required
  • View all active jobs in a table format
  • "Apply" for a job (trigger a confirmation or alert)
  • Search jobs by Title or Skills
  • Delete a job posting from the board
  • Maintain a log of all interactions (post job, search, apply) with timestamps

Non-Functional Requirements:

  • Simple and functional user interface
  • Immediate feedback for user actions
  • Confirmation before deleting any posting
  • Clear display of job details

Implementation Guidelines:

  1. Build using only HTML and Vanilla JavaScript (No CSS allowed)
  2. Use in-memory arrays to store job and application data
  3. Organize the UI into "Client: Post Job" and "Freelancer: Browse Jobs" sections
  4. Include input validation for all fields
  5. Ensure all JavaScript logic is contained within the HTML file

Deliverables:

Create a single HTML file (index.html) that fulfills all the requirements listed above. The application must be functional and demonstrate all CRUD operations clearly.

Marks Distribution (Total: 20 marks):

  • User Interface Structure: 4 marks
  • Job Posting (Add/List): 5 marks
  • Application Functionality: 4 marks
  • Search & Delete Features: 3 marks
  • Validation & Error Handling: 2 marks
  • Activity Logging: 2 marks

Time Limit: 60 minutes

Language: HTML + Vanilla JavaScript (No CSS)
