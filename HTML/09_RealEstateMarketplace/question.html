REAL ESTATE MARKETPLACE

Practical Examination Question (Part C Implementation)

Scenario:

A real estate agency needs a simple software system to manage property listings. Agents should be able to add, view, search, and delete property records. The system should also keep track of all operations performed in an activity log.

Requirements:

Functional Requirements:

  • Add new property with Property ID, Address, Price, and Type (e.g., Apartment, House)
  • View all properties in a table format
  • Search property by its unique ID
  • Delete a property record from the list
  • Basic data validation (ensure all fields are filled)
  • Maintain a log of all actions (add, delete, search) with timestamps

Non-Functional Requirements:

  • Simple and intuitive user interface
  • Fast response for all operations
  • Clear success and error feedback messages
  • Confirmation before deleting any record

Implementation Guidelines:

  1. Build using only HTML and Vanilla JavaScript (No CSS allowed)
  2. Use in-memory arrays to store property and log data
  3. Display property data in a clear table format
  4. Include sections for Add, View, and Search operations
  5. Ensure all JavaScript logic is contained within the HTML file

Deliverables:

Create a single HTML file (index.html) that fulfills all the requirements listed above. The application must be functional and demonstrate all CRUD operations clearly.

Marks Distribution (Total: 20 marks):

  • User Interface Structure: 4 marks
  • Add Functionality: 4 marks
  • View & Search Functionality: 4 marks
  • Delete Functionality: 3 marks
  • Validation & Error Handling: 3 marks
  • Activity Logging: 2 marks

Time Limit: 60 minutes

Language: HTML + Vanilla JavaScript (No CSS)
