COURSE REGISTRATION SYSTEM

Practical Examination Question (Part C Implementation)

Scenario:

A college wants a system to manage courses, students, and course registrations. The admin should be able to add courses/students, register students to courses, and monitor capacity and fee summaries.

Requirements:

Functional Requirements:

  • Add a new course with Course ID, Course Name, Instructor, Capacity, and Fee
  • Add a new student with Student ID, Student Name, Department, and Year
  • Register a student to a selected course with duplicate and capacity checks
  • Drop a registration and free up seats
  • View all courses, students, and registrations in separate tables
  • Search courses by ID/name/instructor and search registrations by Student ID/Course ID
  • Show summary reports: seat availability, enrollment by course, student load, and total fee revenue
  • Maintain activity log with timestamps for all major actions

Non-Functional Requirements:

  • Simple and clearly divided interface using only HTML and JavaScript
  • Input validation and user-friendly success/error messages
  • Immediate UI updates after each operation
  • Persistent data across refresh using browser storage (IndexedDB)

Implementation Guidelines:

  1. Use only HTML + Vanilla JavaScript (No CSS and no external libraries)
  2. Store data in arrays: courses, students, registrations, waitlist, and logs
  3. Use helper functions for rendering tables and computing summaries
  4. Include delete/drop actions in tables where appropriate
  5. Implement login guard using sessionStorage for protected pages

Additional Challenge Questions:

  • Add waitlist handling: if a course is full, registrations should move to waitlist and auto-promote when seats open
  • Add overload validation: prevent students from registering in more than 5 active courses
  • Add department-wise enrollment summary
  • Add conflict check to block registering two courses with the same schedule slot
  • Track dropped registrations in a historical ledger with reason and timestamp

Deliverables:

Create index.html that satisfies all requirements and challenge features with a fully functional interface and logic.

Marks Distribution (Total: 20 marks):

  • UI and Section Structure: 4 marks
  • Course and Student Management: 4 marks
  • Registration and Drop Flow: 4 marks
  • Search + Reporting Features: 4 marks
  • Validation + Logging + Persistence: 4 marks

Time Limit: 60 minutes

Language: HTML + Vanilla JavaScript (No CSS)
