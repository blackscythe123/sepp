E-LEARNING COURSE PLATFORM

Practical Examination Question (Part C Implementation)

Scenario:

An educational platform needs a system to manage courses and student enrollments. Instructors should be able to create courses, and students should be able to enroll in them. The system should track all courses and enrollments.

Requirements:

Functional Requirements:

  • Add a new course with Course ID, Course Name, and Instructor Name
  • View all courses in a table format
  • Enroll a student: Input Student Name and select a Course ID
  • View all enrollments in a separate table
  • Delete a course or an enrollment record
  • Maintain a log of all actions (add course, enroll, delete) with timestamps

Non-Functional Requirements:

  • Simple and clearly divided user interface
  • Immediate confirmation for enrollments
  • Validation to ensure no empty fields
  • Success and error messages for all operations

Implementation Guidelines:

  1. Build using only HTML and Vanilla JavaScript (No CSS allowed)
  2. Use in-memory arrays to store course, enrollment, and log data
  3. Organize the UI into "Instructor: Manage Courses" and "Student: Enrollments" sections
  4. Display results in structured table formats
  5. Ensure all JavaScript logic is contained within the HTML file

Deliverables:

Create a single HTML file (index.html) that fulfills all the requirements listed above. The application must be functional and demonstrate all required features clearly.

Marks Distribution (Total: 20 marks):

  • User Interface Structure: 4 marks
  • Course Management (Add/List): 5 marks
  • Enrollment Functionality: 5 marks
  • Delete Features: 3 marks
  • Validation & Feedback: 1 mark
  • Activity Logging: 2 marks

Time Limit: 60 minutes

Language: HTML + Vanilla JavaScript (No CSS)
