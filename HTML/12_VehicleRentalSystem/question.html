VEHICLE RENTAL SYSTEM

Practical Examination Question (Part C Implementation)

Scenario:

A vehicle rental company needs a system to manage their fleet (cars, bikes) and track rentals. The system should allow staff to add vehicles, rent them to customers, and calculate total costs based on daily rates.

Requirements:

Functional Requirements:

  • Add a new vehicle with Vehicle ID, Model, and Daily Rate (Rs)
  • View all vehicles in a table format with their current status (Available/Rented)
  • Process a rental: Input Vehicle ID and Days, then calculate Total Cost (Days * Daily Rate)
  • Update vehicle status to "Rented" or "Available"
  • Delete a vehicle record from the fleet
  • Maintain a log of all actions (add vehicle, rental, delete) with timestamps

Non-Functional Requirements:

  • Clear and organized user interface
  • Immediate calculation feedback for rentals
  • Confirmation before deleting any record
  • User-friendly messages for invalid IDs or fields

Implementation Guidelines:

  1. Build using only HTML and Vanilla JavaScript (No CSS allowed)
  2. Use in-memory arrays to store vehicle and log data
  3. Include sections for Fleet Management and Rental Processing
  4. Formula for Rental: Total Cost = Number of Days * Vehicle Daily Rate
  5. Ensure all JavaScript logic is contained within the HTML file

Deliverables:

Create a single HTML file (index.html) that fulfills all the requirements listed above. The application must be functional and demonstrate all required features clearly.

Marks Distribution (Total: 20 marks):

  • User Interface Structure: 4 marks
  • Fleet Management (Add/List): 5 marks
  • Rental Processing & Calculation: 5 marks
  • Status Updates & Delete: 3 marks
  • Validation & Feedback: 1 mark
  • Activity Logging: 2 marks

Time Limit: 60 minutes

Language: HTML + Vanilla JavaScript (No CSS)
