## 📊 Exam Structure Breakdown

```
Part A: Requirements Analysis (15 marks, 30 min)
├── Identify Functional Requirements (What system does)
├── Identify Non-Functional Requirements (How it behaves)
└── List specific, clear requirements

Part B: System Design (20 marks, 30 min)
├── DFD (Data Flow Diagram) OR
├── Activity Diagram
└── Show system flow, processes, data stores

Part C: Implementation (20 marks, 60 min)
├── Run one of the provided examples OR
├── Build from HTML/Streamlit template
├── Implement basic CRUD operations
└── Test thoroughly

Part D: Git & Tools (20 marks, 60 min)
├── Initialize Git repository
├── Make meaningful commits
├── Show version control understanding
└── Explain branching/workflow

Part E: Learning Outcomes (5 marks, flexible)
└── Write brief summary of learning
```

## 💡 Study Tips

1. **Read the Examples First**
   - Look at `EXAM_ANSWER_EXAMPLE.md` to see exact format expected

2. **Practice with Templates**
   - Use DFD template from `DFD_GUIDE.md`
   - Use implementation templates from `PART_C_IMPLEMENTATION_GUIDE.md`

3. **Adapt Not Copy**
   - Take one system and modify for different scenario
   - Create your own requirements list
   - Design your own diagrams

4. **Test Your Code**
   - Run the examples first
   - Modify one feature
   - Test it works

5. **Understand Not Memorize**
   - Know WHY each requirement matters
   - Understand WHAT DFD shows
   - Learn HOW to implement CRUD
