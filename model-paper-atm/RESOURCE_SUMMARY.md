## 🎯 What Each Part Tests

### Part A: Analytical Skills (15 marks)
- Can you read and understand requirements?
- Can you identify "What" (Functional) vs "How" (Non-Functional)?
- Are your requirements specific and measurable?

### Part B: Design Skills (20 marks)
- Can you visualize system flow?
- Do you understand DFD symbols and data flow?
- Can you show system components and their interactions?

### Part C: Implementation Skills (20 marks)
- Can you build a working system from scratch?
- Is your code clean and understandable?
- Do you handle validation and errors?

### Part D: Professional Skills (20 marks)
- Do you understand version control?
- Can you work with Git and make meaningful commits?
- Do you follow software engineering practices?

### Part E: Reflection (5 marks)
- Can you articulate what you learned?
- Do you understand practical applications?

## 💡 Key Success Factors

✓ **Understand, don't memorize**
- Know WHY each requirement matters
- Know WHAT DFD shows, not just symbols
- Know HOW to build CRUD, not just copy-paste

✓ **Practice with examples**
- Modify existing systems
- Create requirements for new scenarios
- Draw DFDs for different domains

✓ **Focus on simplicity**
- Simple, working code > Complex broken code
- Clear, labeled diagrams > Pretty but unclear diagrams
- Specific requirements > Vague requirements

✓ **Manage time effectively**
- Don't spend 2 hours on Part C
- Don't make complex UI in HTML
- Don't overcomplicate requirements
